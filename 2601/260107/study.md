# 캐시 스탬피드
캐시 항목이 만료되는 순간 다수의 요청이 동시에 캐시를 갱신하려고 하면서 발생하는 문제

### 해결 방법
1. 잠금 방식: 한 요청 스레드가 캐시 키에 대한 잠금을 획득. 성능 저하 가능성이 존재한다.
2. 외부 재계산 방식: 캐시를 주기적으로 모니터링하는 스레드를 별도로 관리하여 캐시를 갱신한다. 메모리를 비효율적으로 사용할 가능성이 존재한다.
3. PER 알고리즘 방식: 확률 개념을 사용하여 캐시 만료 시간이 얼마 남지 않았을 경우 캐시를 갱신하는 방법.

# HTTP
HTTP는 웹상에서 클라이언트와 서버 간 통신을 위한 프로토콜입니다.
### HTTP 1.0
한 개의 요청과 응답마다 TCP 커넥션을 생성하여 사용됐습니다. 하지만 이 방식은 매 요청마다 연결을 생성하는 오버헤드가 발생합니다.
### HTTP 1.1
이러한 문제를 지속 커넥션이라는 지정한 타임아웃만큼 커넥션을 종료하지 않는 방식으로 해결합니다.

또한, 파이프라이닝을 지원하여 요청의 응답 지연을 감소합니다. 파이프라이닝에서 HTTP 요청은 연속적이며, 순차적으로 전달됩니다.

하지만 Head of Line Blocking (HOLB) 문제가 있습니다. 첫번째 요청이 오래 걸리게 되면 나머지 요청들이 지연되는 문제입니다.

### HTTP 2.0
1.1은 HTTP 메세지를 텍스트 형식으로 전송했지만, 2.0부터는 프레임 단위로 분할하고 바이너리 형태로 전송합니다. 따라서 파싱 및 전송 속도가 향상되었습니다.

또한, 멀티플렉싱을 지원합니다. 하나의 커넥션을 통해 요청과 응답을 병렬로 처리할 수 있는 방식입니다. 각 요청이 독립적으로 처리되어 HOLB 문제를 해결합니다. HPACK 헤더 압축 방식을 사용해 반복되는 헤더를 효율적으로 관리하여 대역폭 사용이 최적화되었습니다.
