# 관계형 데이터베이스와 비 관계형 데이터베이스의 차이점은 무엇인가요?

관계형 데이터베이스는 고정된 로우와 컬럼으로 구성된 테이블에 데이터를 저장합니다. 테이블 간에 조인을 통해 테이블을 합쳐서 조회가 가능합니다. 데이터를 중복 없이 한번만 저장을 하고, 데이터 무결성을 보장합니다. 단점으로는 한번 정한 스키마를 유연하게 변경하기 어렵습니다. 또한, 복잡한 쿼리와 성능에 주의하여야 합니다.

비관계형 데이터베이스는 정해진 스키마가 없이 자유롭게 데이터를 저장하고 조회합니다. 도큐멘트, 키-밸류, 그래프 유형 등이 존재합니다. 대량의 데이터와 높은 사용자 부하에서 유리한 편입니다. 반면 중복이 가능하여 데이터 일관성에 주의해야 합니다.

### 어떤 상황에서 각 유형의 데이터베이스를 사용할까?
데이터가 구조화되어 있고, 트랜잭션, 데이터 무결성과 데이터 일관성이 중요한 경우 관계형 데이터베이스를 사용합니다.

낮은 응답시간이 요구되거나, 데이터의 스키마가 빈번히 변경되거나, 아주 많은 양의 데이터를 저장해야 하는 경우에는 비 관계형 데이터베이스가 유리합니다.
