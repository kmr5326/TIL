# CPU 스케줄링
CPU 스케줄링은 운영체제가 프로세스들에게 공정하고 합리적으로 CPU 자원을 배분하는 것을 의미합니다.

- 선점형 스케줄링: 프로세스가 CPU를 사용하고 있더라도 운영체제가 자원을 빼앗아 다른 프로세스에 할당할 수 있는 방식. 응답 시간이 낮으며, 효율적이지만 컨텍스트 스위칭 오버헤드가 발생할 수 있으며 경쟁 상태가 발생할 수 있습니다.
- 비선점형 스케줄링: 하나의 프로세스가 자원을 사용하고 있다면 해당 프로세스가 종료되거나 스스로 대기 상태로 전환되기 전까지 다른 프로세스가 자원을 점유할 수 없는 스케줄링 방식입니다. 컨텍스트 스위칭 비용이 상대적으로 적지만, 응답 시간이 길 수 있습니다.

### CPU 스케줄링 알고리즘
- 선입 선처리 스케줄링
- 최단 작업 우선 스케줄링
- 라운드 로빈 스케줄링
- 최소 잔여 시간 우선 스케줄링
- 다단계 큐 스케줄링, 다단계 피드백 큐 스케줄링(에이징)

### 기아 현상 해결법
에이징 기법을 적용한다. 에이징이란 오랫동안 대기한 프로세스의 우선순위를 점차 높이는 방식입니다.
